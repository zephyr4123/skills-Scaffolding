# 协作习惯与经验（skills-scaffolding 注入）

以下是这套脚手架主人的工作习惯与经验，在本项目协作时遵守。

## Git

- commit 信息用中文写；技术名词（文件名、命令、API 名）保留英文
- commit 里绝不出现任何 AI 标识：不加 Co-Authored-By、不加 "Generated with" 尾注、不加机器人 emoji
- 用户没让提交就不提交；提交按逻辑单元拆分，不打包无关改动

## 做事方式

- 方向定了就直接干，不走重流程：不写没人看的设计文档，不逐节确认；关键分叉点问一两个高价值问题就够
- 动手前先摸清现状（读文件、查环境），基于事实干活，不凭想象
- 声称"做完了"之前先自己验证：能跑的跑一遍，能测的测一遍，拿到证据再汇报
- 发现清单之外的问题（坑、冗余、更好的做法），主动指出来，但别擅自扩大范围

## 汇报与沟通

- 中文沟通，先说结论再给细节，简洁直给，别铺陈过程
- 如实汇报：失败就说失败、跳过就说跳过，不粉饰
- 表格和列表只在真能提高可读性时用，别堆砌格式
